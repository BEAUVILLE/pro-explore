Ordre de circulation conseillé

Chemin simple pour travailler proprement sans se disperser.
1. Entrée PIN Passer par pin.html pour ouvrir la session pro.
2. Cockpit Lire l’état du terrain dans cockpit.html.
3. Lieux Vérifier les fiches existantes dans lieux.html.
4. Submit Créer ou corriger une fiche dans submit.html.
5. Vérification publique Contrôler la lecture finale dans la vitrine publique et fiche.html.

Doctrine DIGIY

Le moule maître à garder pour éviter les dérapages.
Toujours partir du rail réel validé. Ne pas inventer un backend parallèle.
Zéro démo si le backend existe déjà.
Ne pas mélanger ancien rail et nouveau rail dans un même fichier.
Navigation croisée visible entre les portes principales du module.
Lecture simple côté terrain, pas d’interface laboratoire.
Travailler fichier par fichier, proprement, sans casser le reste.
